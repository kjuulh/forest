# Configuration Files

Reference for all file types used by Forest.

## Project Files

### `forest.cue`

The main project configuration file. Located at the project root.

```cue
package my_service

import "forest.sh/forest/sdk@v0"

project: sdk.#ForestProject & {
    name:         "my-service"
    organisation: "my-org"
}

dependencies: sdk.#ForestDependencies & {
    "org/component": version: "0.1"
}

"org": "component": sdk.#ForestComponentUsage & {
    env: { /* per-environment config */ }
    config: { /* shared config */ }
}

commands: sdk.#ForestProjectCommands & {
    dev: ["cargo run"]
}
```

### `forest.lock`

Lock file for resolved dependencies. Generated by `forest add` and `forest update`. Do not edit manually.

```
# forest.lock — do not edit manually
forest-contrib/kubernetes-service@0.1.2 linux/amd64 sha256:abc123...
forest-contrib/terraform-service@0.1.0 path:../../components/forest-contrib/terraform-service
```

**Format:** One dependency per line:

- Registry: `org/name@version os/arch sha256:hash`
- Path: `org/name@version path:relative/path`

### `cue.mod/module.cue`

CUE module metadata. Required for CUE evaluation.

```cue
module: "forest.sh/my-org/my-service@v0"
language: version: "v0.12.0"
deps: {
    "forest.sh/forest/sdk@v0": v: "0.1.0"
}
```

## Component Files

### `forest.cue` (Component)

Component metadata — name, version, build and codegen settings.

```cue
package my_component

import "forest.sh/forest/sdk@v0"

component: sdk.#ForestComponent & {
    name:    "my-component"
    version: "0.1.0"
    upload: { type: "rust", source: "./crates/my-component" }
    codegen: { type: "rust", output: "./crates/my-component/src/" }
}
```

### `forest.component.cue`

Component spec schema, commands, and hooks. This is the component's public API.

```cue
package my_component

import "forest.sh/forest/sdk@v0"

#Spec: sdk.#ForestSpec & { /* schema */ }
#Commands: sdk.#ForestCommands & { /* commands */ }
#Hooks: sdk.#ForestHooks & { /* hooks */ }
```

### `spec.cue` (Optional)

Additional CUE type definitions imported by `forest.component.cue`. Useful for separating large type hierarchies.

## Generated Files

### `forestgen.rs` / `forestgen.ts`

Generated SDK code from `forest generate`. Contains typed structs matching the component spec, input/output types for commands, and method routing. Regenerated on each `forest generate` run.

### `.forest/component/meta.json`

Build metadata cached after `forest build`. Contains binary hashes per platform and the cached component descriptor.

## Server Configuration

### `.env`

Local development environment variables:

```env
DATABASE_URL=postgresql://devuser:devpassword@localhost:5432/dev
NATS_URL=nats://localhost:4222
FOREST_SERVICE_ACCOUNT_API_KEY=<optional>
```

### `docker-compose.yaml`

Located at `templates/docker-compose.yaml`. Starts PostgreSQL and NATS for local development:

```bash
mise run local:up    # Start services
mise run local:down  # Stop and clean up
```

## Cache Locations

| Path | Contents |
|------|----------|
| `~/.cache/forest/components/bin/` | Content-addressable binary cache |
| `~/.config/forest/` | Authentication credentials |
| `.forest/` | Project-local cache (build metadata) |

## Templates

Component templates live in `templates/deployment/{destination_type}/`:

```
templates/
  deployment/
    forest/kubernetes@1/
      deployment.yaml
      service.yaml
    forest/terraform@1/
      main.tf
      variables.tf
    forage/containers@1/
      .gitkeep
```

Templates are rendered during `forest release prepare` with the project's spec values and copied to the working directory.
